# UI 디자인 수정 - 브랜드 컬러 및 헤더 리디자인

## 1. 디자인 목표

### 사용자 경험 목표
- **브랜드 일관성**: 든든팀의 메인 컬러 #672091을 전체 UI에 일관되게 적용
- **직관성 향상**: "AI 새벽배송" 콘셉트를 명확하게 전달
- **사용성 개선**: 폼 라벨을 더 명확한 문구로 변경하여 사용자 이해도 향상

### 시각적 목표
- 따뜻하고 친근한 보라색 톤으로 브랜드 정체성 강화
- 새벽 배송 콘셉트를 시각적으로 표현 (🌙, 🚚 이모지 활용)
- 글래스모피즘 스타일 유지하면서 가독성 향상

### 브랜드 정렬
- 든든팀 브랜드 컬러 #672091 중심
- "새벽배송" 콘셉트 반영
- 따뜻하고 신뢰감 있는 이미지

## 2. 사용자 플로우

### 주요 사용자 경로
1. 랜딩 → 헤더에서 "AI 새벽배송" 콘셉트 인지
2. "새 배송지 등록" 버튼 클릭 → 사례 등록 모달 오픈
3. 폼 작성 (명확해진 라벨로 쉽게 이해)
4. 등록 완료

### 개선 사항
- 헤더 타이틀이 더 명확하고 친근하게 변경
- 서브카피로 가치 제안 명확화
- 버튼 문구가 "새 배송지 등록"으로 변경되어 콘셉트 일관성 유지

## 3. 화면 설계

### 3.1 헤더 (Header)

#### 변경 전
- 타이틀: "✨ PM AI Knowledge Hub"
- 서브카피: "AI 활용 사례를 공유하고 배워보세요"
- 버튼: "+ 새 사례 등록"

#### 변경 후
- 타이틀: "🌙 AI 새벽배송 🚚"
- 서브카피: "오늘도 지식이 배송됩니다"
- 버튼: "+ 새 배송지 등록"

#### 디자인 요소
- 타이틀: 3xl (텍스트 크기), 볼드, 이모지 포함
- 서브카피: 작은 텍스트, 보라색 톤 (text-purple-200)
- 버튼: #672091 배경, 흰색 텍스트, 둥근 모서리

### 3.2 등록 폼 (Add Case Modal)

#### 변경 전
- "💡 왜 했는지?"
- "🛠️ 무엇이 좋아졌는지?"
- "✨ 어떤 결과로 이어졌는지?"

#### 변경 후
- "💡 왜 했는지?" (유지)
- "🛠️ 어떻게 했는지?" (변경)
- "✨ 어떤 결과로 이어졌는지?" (유지)

#### 플레이스홀더 개선
- 왜: "어떤 문제나 필요 때문에 AI를 활용했나요?"
- 어떻게: "구체적으로 어떤 방법으로 활용했나요?"
- 결과: "최종적으로 어떤 성과나 변화가 있었나요?"

## 5. 반응형 디자인

### Desktop (1280px+)
- 헤더 높이: 충분한 패딩 (py-6)
- 타이틀과 버튼이 좌우 배치 (flex justify-between)
- 최대 너비: 7xl (max-w-7xl)

### Tablet (768px - 1279px)
- 헤더 레이아웃 유지
- 약간 줄어든 패딩 (sm:px-6)

### Mobile (< 768px)
- 헤더 세로 정렬 고려 (필요시)
- 버튼 전체 너비 (필요시)
- 최소 패딩 (px-4)

## 6. 접근성

### 색상 대비
- 흰색 텍스트 on #672091: 7.3:1 (WCAG AAA)
- 흰색 텍스트 on 반투명 배경: 4.8:1 (WCAG AA)
- Purple 200 on 어두운 배경: 4.5:1 (WCAG AA)

### 키보드 네비게이션
- Tab으로 모든 버튼 및 입력 필드 접근 가능
- Enter로 버튼 실행
- Esc로 모달 닫기

### 스크린 리더
- 이모지 후 의미 있는 텍스트 제공
- 버튼에 명확한 라벨
- 폼 필드에 적절한 label 연결

## 12. 디자인 의도

### 왜 #672091인가?
- 든든팀의 브랜드 아이덴티티 컬러
- 기존 보라색보다 더 따뜻하고 신뢰감 있는 톤
- 새벽 배송 콘셉트와 잘 어울리는 깊은 보라색

### 왜 "AI 새벽배송"인가?
- 지식이 매일 아침 배송되는 콘셉트
- 사용자에게 친근하고 기억하기 쉬운 브랜딩
- 이모지(🌙, 🚚)로 콘셉트 강화

### 왜 "어떻게 했는지?"로 변경했는가?
- "무엇이 좋아졌는지?"는 결과와 중복
- "어떻게"는 방법론을 명확히 묻는 질문
- Why → How → Result 구조가 더 논리적
